# Report Template

Use this structure for every review. Fill in, don't skip sections (mark "None found" if empty).

```markdown
## UI/UX Review: [Component/Page name]

### Summary
[2-3 sentences: overall read, strongest point, biggest risk]

### Findings

**Blocker** (WCAG failure or blocks task completion)
- [Issue] — [WCAG SC ref if applicable] — [Concrete fix]

**Major** (significant friction, not blocking)
- [Issue] — [Fix]

**Minor** (polish-level)
- [Issue] — [Fix]

**Suggestions** (nice-to-have / creative direction)
- [Idea] — [Why it'd help]

### Scorecard

| Axis | Rating | Note |
|---|---|---|
| Layout | ✅/⚠️/❌ | |
| Accessibility | ✅/⚠️/❌ | |
| Contrast | ✅/⚠️/❌ | |
| Responsiveness | ✅/⚠️/❌ | |
| User flow | ✅/⚠️/❌ | |
| Design consistency | ✅/⚠️/❌ | |
| Visual design / theme | ✅/⚠️/❌ | |

### Global readiness (only if worldwide/multi-region audience relevant)
[Note any i18n/l10n/regional-standard flags, or "Not assessed — single-region scope confirmed"]
```

Rating legend: ✅ solid, ⚠️ needs work, ❌ fails / needs redesign.
